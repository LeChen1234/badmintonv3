# 羽毛球训练动作标注系统 - 功能总结

## 一、已有功能

### 1. 用户与认证
| 功能 | 说明 |
|------|------|
| **公开注册** | 支持用户自助注册（可配置关闭），注册后角色为学生 |
| **登录 / 登出** | JWT 认证，Token 存 localStorage |
| **当前用户信息** | `/api/auth/me` 获取当前登录用户 |
| **角色与权限** | 四类角色：管理员(admin)、专家(expert)、组长(leader)、学生(student)，按角色控制接口与菜单 |

### 2. 用户管理（管理员）
| 功能 | 说明 |
|------|------|
| 用户列表 | 按角色筛选、分页 |
| 新建用户 | 设置用户名、密码、显示名、角色 |
| 编辑用户 | 修改显示名、角色、密码、启用状态 |
| 禁用用户 | 软删除（is_active=false） |

### 3. 项目管理（管理员/专家）
| 功能 | 说明 |
|------|------|
| 项目列表 | 分页查询 |
| 新建项目 | 名称、描述 |
| 编辑/删除项目 | 修改信息或删除项目 |

### 4. 任务管理
| 功能 | 说明 |
|------|------|
| 任务批次列表 | 按项目、负责人、状态筛选；学生仅看自己的任务 |
| 新建任务批次 | 关联项目、名称、动作类别、总帧数、负责人、截止日期 |
| 编辑/分配任务 | 修改批次信息、分配给学生 |
| **上传图片/视频** | 学生或管理员可为任务上传多张图片或一个视频，视频自动抽帧（最多 500 帧） |
| **获取帧列表/帧图** | 按帧序号获取该任务下的图片，用于标注页展示 |
| 大模型初标（可选） | 配置开启后，可触发 ML 初标，默认关闭 |

### 5. 标注（内建）
| 功能 | 说明 |
|------|------|
| **标注页** | 按任务进入，无帧时先上传图片/视频，有帧后逐帧标注 |
| **帧展示与导航** | 显示当前帧图片，上一帧/下一帧/输入帧号跳转 |
| **关键点标注（骨架）** | 25 点人体+球拍；画布叠在图片上，选点打点、点击已有点选中、拖拽微调；清除当前/全部 |
| **算法辅助标注（多人）** | 当前帧姿态估计（MediaPipe full/heavy/lite），多人时可选「第 N 人」应用骨架；球拍两点人工补标 |
| **标注表单** | 动作类型、动作阶段、动作质量、备注 |
| **标注可视化** | 当前帧上叠加骨架与关键点、动作类型、阶段、质量、标注人、状态 |
| **保存/更新标注** | 每帧可保存或更新，带标注人 ID 与姓名、keypoints 持久化 |
| **提交草稿** | 将当前任务下所有草稿标注提交为「已提交」 |
| **确认标注** | 管理员/专家/组长将「已提交」标注确认为「已确认」 |
| 批量创建标注 | API 支持按帧批量写入 |

### 6. 审核流程
| 功能 | 说明 |
|------|------|
| 提交审核 | 标注员将任务提交自核/组长/专家审核 |
| 通过/打回 | 组长/专家可通过或打回，打回可填原因 |
| 审核历史 | 查看某任务的审核记录时间线 |

### 7. 进度监控
| 功能 | 说明 |
|------|------|
| 仪表盘 | 项目总数、任务批次、已完成锁定数、整体完成率；各项目进度表、人员工作量表 |
| 进度页 | 整体完成率仪表盘、各项目进度条、人员工作量统计 |

### 8. 数据导出
| 功能 | 说明 |
|------|------|
| **确认数据集导出** | 按项目导出所有「已确认」标注到单个文件，支持 **JSON / COCO / CSV** 三种格式 |
| **每帧带标注人+关键点** | 导出内容含 annotator_id、annotator_name、keypoints（25 点）、动作类型/阶段/质量等；COCO 含 keypoints 数组 |
| 下载导出文件 | 按项目+文件名下载已生成的导出文件 |
| 已确认数量 | 查询某项目下已确认标注条数 |

### 9. 技术特性
| 项目 | 说明 |
|------|------|
| 数据库 | SQLite（单文件），无需单独安装 PostgreSQL |
| 存储 | 数据与上传文件均在项目目录（如 D:\badminton\data） |
| 部署 | 无 Docker，本地运行；后端 `0.0.0.0`，前端 `host: 0.0.0.0`，支持局域网访问 |
| 依赖安装 | 支持一键脚本，依赖可全部安装在 D 盘（venv + node_modules） |

### 10. 已实现的优化（近期补充）
| 功能 | 说明 |
|------|------|
| **任务 completed_frames 同步** | 确认标注后自动按「已确认」数量更新任务批次的已完成帧数 |
| **审计日志** | 确认标注、导出项目、提交/通过/打回审核时写入 AuditLog，便于追溯 |
| **标注页快捷键** | ← 上一帧、→ 下一帧、Ctrl+S 保存 |
| **GET /api/config** | 前端可读 allow_public_register、ml_backend_enabled；登录页根据配置隐藏注册入口 |
| **多格式导出** | 已支持 JSON、COCO、CSV 三种格式 |
| **视频抽帧可配置** | 上传接口支持 max_frames 参数（1–2000，默认 500） |
| **使用说明** | docs/使用说明.md 已更新为 SQLite + 内建标注 + 上传流程 |

## 二、仍缺少或可增强的功能

### 1. 功能缺失
| 项目 | 说明 |
|------|------|
| ~~**关键点标注**~~ | 已实现：25 点骨架画布、选点打点/拖拽、算法辅助多人识别、保存与导出含 keypoints |
| **任务状态自动推进** | 未实现「全部帧已确认后自动将任务设为已锁定」等状态联动 |
| **视频按秒/按间隔抽帧** | 目前仅有 max_frames 上限，无「每 N 秒一帧」或「每 N 帧取一帧」等策略 |
| **忘记密码 / 重置密码** | 无找回密码、邮箱验证或管理员重置入口 |
| **审计日志查询界面** | 有写入无查询，管理端无法查看操作日志列表 |

### 2. 体验与健壮性
| 项目 | 说明 |
|------|------|
| **大文件上传** | 视频 500MB、单图 20MB 限制，无分片、断点续传 |
| **标注页无图时的引导** | 可增加示例数据或「从模板创建任务」引导 |
| **移动端适配** | 未针对触屏、小屏做响应式布局 |
| **错误重试与提示** | 可增加重试按钮与更明确的错误说明 |
| **导出进度** | 大批量导出时无进度条或后台任务查询 |

### 3. 运维与配置
| 项目 | 说明 |
|------|------|
| **配置界面** | 公开注册、大模型开关等仅能改 .env，无管理端配置页 |
| **数据备份与恢复** | 无一键备份/恢复 SQLite 与 data 目录的脚本或界面 |

### 4. 可选增强
| 项目 | 说明 |
|------|------|
| **标注模板** | 预置常用「动作类型+阶段」组合，一键填充 |
| **批量审核** | 审核页多选任务后批量通过/打回 |
| **统计图表** | 仪表盘/进度页按时间、按人的趋势图（ECharts 已引入） |
| **VLM 格式导出** | 若需训练视觉语言模型，可再增加 VLM 格式导出 |

## 三、总结

- **已有**：注册/登录、用户与角色、项目与任务、图片/视频上传与选帧（含 max_frames）、**关键点标注（25 点骨架、画布叠图、选点/拖拽、算法辅助多人识别）**、内建标注与可视化、快捷键、提交与确认、审核流程、进度统计、多格式导出（JSON/COCO/CSV，含 keypoints）、审计日志写入、completed_frames 同步、config 接口、使用说明更新、D 盘依赖。
- **仍缺/可加强**：任务状态自动推进、视频抽帧策略细化、忘记/重置密码、审计日志查询页、大文件上传与移动端、配置界面、数据备份恢复、标注模板、批量审核、统计图表等。

如需优先实现某一类（如「任务状态自动推进」或「审计日志查询」），可在此基础上再拆成具体开发任务。
