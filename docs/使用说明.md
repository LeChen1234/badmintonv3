# 羽毛球标注平台 - 使用说明与演示

本文档演示如何从零启动并完整使用整个项目。

---

## 一、环境准备

- 已安装：Python 3.10+、PostgreSQL 16、Redis，可选 MinIO
- 前端：Node.js 18+

---

## 二、一键快速开始（推荐）

在项目根目录执行：

```bash
# 1. 复制样例配置（Windows）
copy examples\env.minimal.sample .env

# 或 Linux/Mac
# cp examples/env.minimal.sample .env

# 2. 编辑 .env，至少修改：
#    LABEL_STUDIO_API_KEY — 见下方「获取 API Key」步骤

# 3. 安装依赖并执行数据库迁移（Windows PowerShell）
.\scripts\install_and_run.ps1

# 4. 启动后端与前端（见「五、本地开发」），然后获取 Label Studio API Key 并写入 .env，初始化平台
python scripts/init_platform.py

# 5. 生成测试数据
python scripts/generate_mock_data.py
```

### 获取 Label Studio API Key

1. 浏览器打开：http://localhost:8080  
2. 使用 `.env` 中的 `LABEL_STUDIO_USERNAME` / `LABEL_STUDIO_PASSWORD` 登录  
3. 右上角头像 → Account & Settings → Access Token → Copy Token  
4. 将 Token 填入 `.env` 的 `LABEL_STUDIO_API_KEY`，再重新执行 `python scripts/init_platform.py`

## 四、完整使用流程演示

### 1. 登录管理前端

1. 打开 http://localhost:3000  
2. 使用 **expert1** / **expert123** 登录  
3. 可查看仪表盘、用户列表、项目与任务批次  

### 2. 在 Label Studio 中标注

1. 打开 http://localhost:8080  
2. 使用同一套账号或 LS 本地账号进入  
3. 选择初始化脚本创建的项目，导入 `generate_mock_data.py` 生成的数据（或通过管理后端分配的任务）进行标注  

### 3. 调用后端 API

- 使用项目自带示例：打开 **examples/api-examples.http**，用 VS Code REST Client 依次执行「登录」→「获取当前用户」等请求。  
- 或使用 curl：
  ```bash
  TOKEN=$(curl -s -X POST http://localhost:8000/api/auth/login \
    -H "Content-Type: application/json" \
    -d "{\"username\":\"expert1\",\"password\":\"expert123\"}" | jq -r .access_token)
  curl -H "Authorization: Bearer $TOKEN" http://localhost:8000/api/auth/me
  ```

### 4. 导出数据

- 管理前端：登录后进入「导出」页面，选择批次与格式（Label Studio JSON、COCO JSON、CSV、VLM JSON）导出。  
- API：如 `GET /api/export/csv?batch_id=1`（需在请求头带 `Authorization: Bearer <token>`）。

## 五、本地开发

1. 确保 PostgreSQL、Redis（及可选 MinIO、Label Studio、ML Backend）已在本机启动，`.env` 中主机名为 `localhost`。
2. 后端：在项目根目录确保有 `.env` 并已执行过 `.\scripts\install_and_run.ps1`，然后：
   ```bash
   cd backend && ..\.venv\Scripts\python.exe -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
   ```
3. 前端：  
   ```bash
   cd frontend && npm install && npm run dev
   ```
4. 仍按上文执行 `init_platform.py` 与 `generate_mock_data.py`（注意 `.env` 中 `LABEL_STUDIO_HOST` 等为 `http://localhost:8080`）。

## 六、常见问题

- **init_platform 报 401**：未设置或填错 `LABEL_STUDIO_API_KEY`，请按「获取 Label Studio API Key」步骤操作。  
- **后端连不上数据库**：确保 PostgreSQL、Redis 已启动，检查 `.env` 中 `POSTGRES_HOST`、`REDIS_HOST` 为 `localhost`。  
- **前端 3000 访问 API 报错**：确认后端已启动且 `BACKEND_CORS_ORIGINS` 包含 `http://localhost:3000`。  

更多配置说明见根目录 **.env.example** 与 **examples/env.minimal.sample**。
