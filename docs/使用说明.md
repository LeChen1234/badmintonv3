# 羽毛球标注平台 - 使用说明

本文档说明如何从零启动并完整使用当前版本（SQLite + 内建标注 + 图片/视频上传）。

---

## 一、环境准备

- **Python** 3.9+（推荐 3.10-3.13，需加入 PATH）
- **Node.js** 18+（含 npm，需加入 PATH）
- 无需安装 PostgreSQL、Redis、Docker

---

## 二、一键运行（推荐）

所有依赖与数据均在当前项目目录完成（推荐放在本地非系统盘路径）。

在项目根目录双击运行：

```
run.bat
```

脚本将：

1. 在项目根目录 `.venv` 创建/使用 Python 虚拟环境并安装后端依赖  
2. 在 `frontend\node_modules` 安装前端依赖  
3. 创建 `data`、`data\uploads`、`data\exports`  
4. 新开两个窗口分别启动 **后端**（8000）与 **前端**（3000）

---

## 三、访问地址与默认账号

| 服务     | 地址                  | 说明                     |
|----------|-----------------------|--------------------------|
| 管理前端 | http://localhost:3000 | 本机登录、任务、标注、导出等 |
| 后端 API | http://localhost:8000 | 健康检查：/api/health    |

**让其他人也能用：**

- **同一 WiFi / 同一局域网**：运行 `run.bat` 后，窗口会显示局域网访问提示。把地址发给同事/同学即可访问。若打不开，请在本机 Windows 防火墙中允许 Node、Python 通过。
- **不在同一网络（如家里/公司）**：需要把服务放到公网，例如：本机用内网穿透（ngrok、frp 等）暴露 3000 端口；或把前后端部署到一台有公网 IP 的服务器上（见下方「七、常见问题」）。

**默认管理员账号（首次启动自动创建）：**

| 用户名 | 密码     | 角色   |
|--------|----------|--------|
| admin  | admin123 | 管理员 |

其他用户可通过登录页 **注册** 或由管理员在「用户管理」中创建。

## 四、使用流程

### 1. 登录与注册

1. 打开 http://localhost:3000  
2. 新用户可点击「注册」填写用户名、显示名、密码  
3. 使用管理员 **admin / admin123** 或已注册账号登录  

### 2. 创建项目与任务（管理员/专家）

1. 进入 **任务与标注** → 可选先建「项目」（若使用项目管理）  
2. 点击 **新建任务批次**：选择项目、填写批次名称、总帧数可填 0（由上传决定）、分配负责人  
3. 负责人（学生）在任务列表中点击 **标注** 进入标注页  

### 3. 上传图片或视频（学生/负责人）

1. 进入 **标注** 页后，若该任务尚无帧，会显示上传区  
2. 拖拽或选择 **多张图片**（jpg/png/gif 等）或 **一个视频**（mp4/avi/mov 等）  
3. 点击「开始上传」：图片按张数生成帧，视频自动抽帧（最多 500 帧，可在后续版本中配置）  
4. 上传完成后自动进入逐帧标注界面  

### 4. 逐帧标注

1. 左侧显示当前帧图片，右侧为标注表单  
2. **关键点标注（骨架图）**：标注叠加在图片上。右侧列表选择某一关键点（如左肩、右腕），在画布上**点击**即可在该位置打点；**点击画布上已有蓝点**会选中该节点（右侧列表高亮），可再**拖拽**微调。支持「清除当前点」「清除全部」。  
3. **算法辅助标注（多人识别）**：点击「算法辅助标注（多人识别）」按钮，后端会对当前帧做人体姿态估计（MediaPipe），支持**多人检测**。若检测到多人会弹出选择框，选择「第 N 人」后应用该人的骨架；若仅一人则直接应用。人体 23 点由算法生成，球拍两点（拍柄、拍头）需人工补标。生成后可继续在画布上微调或补点，再保存。首次使用多人检测时会自动下载模型到 `data/models/`（D 盘）。  
4. 选择 **动作类型**（杀球/高远球/吊球等）、**动作阶段**（准备/引拍/击球等）、**质量**、备注  
5. 点击「保存标注」或「保存并下一帧」；每帧会记录 **标注人**  
6. 画面上会叠加显示当前骨架与关键点，便于核对  
7. 支持 **快捷键**：← 上一帧、→ 下一帧、Ctrl+S 保存  

### 5. 提交与确认

1. 标注员完成一批后点击 **提交所有草稿**，状态变为「已提交」  
2. 管理员/专家/组长在标注页或审核流程中 **确认所有已提交标注**，状态变为「已确认」  

### 6. 导出已确认数据

1. 进入 **数据导出**，选择项目  
2. 选择格式：**JSON**（含标注人）/ **COCO** / **CSV**  
3. 点击「导出已确认数据集」后，可下载单个文件（所有已确认标注）

## 五、本地开发（手动启动）

1. **后端**：
   ```bash
   cd backend
   ..\.venv\Scripts\python.exe -m uvicorn app.main:app --host 0.0.0.0 --port 8000
   ```
2. **前端**：
   ```bash
   cd frontend
   npm install
   npm run dev
   ```
3. 前端默认代理 `/api` 到 `http://localhost:8000`，支持局域网访问（host: 0.0.0.0）。  

---

## 六、配置说明

- 配置文件：项目根目录 **.env**（可选）  
- 数据库：SQLite，文件位于 `data/badminton.db`  
- 上传文件：`data/uploads/`  
- 导出文件：`data/exports/`  
- 常用项：
  - `ALLOW_PUBLIC_REGISTER=true`：是否允许公开注册  
  - `ENABLE_ML_BACKEND=false`：是否启用大模型初标（需自建 ML 服务）  

---

## 七、常见问题

- **前端访问 API 报错**：确认后端已启动，且浏览器访问的地址与前端代理一致（如 localhost:3000 代理到 localhost:8000）。  
- **上传失败**：检查 `data/uploads` 目录是否存在、磁盘空间是否充足；视频建议不超过 500MB。  
- **一键启动**：在项目根目录执行 `run.bat`（或双击 `run_all.bat`）。  

更多功能说明见 **docs/项目功能总结.md**。
