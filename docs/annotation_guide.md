# 羽毛球训练动作标注规范

## 1. 标注概述

本系统用于标注羽毛球训练视频帧，包含两类标注任务：

- **骨架关节点标注**：标注运动员的 25 个身体关键点
- **动作分类标注**：标注当前帧的动作类型、动作阶段和动作质量

### 2.2 标注原则

1. **关节中心**：所有关节点应标注在关节的几何中心
2. **遮挡处理**：
   - 部分遮挡：根据可见部分推测关节位置，标注在推测位置
   - 完全遮挡：跳过该关节点，不进行标注
3. **左右判定**：以图片中运动员自身的左右为准（镜像原则）
4. **球拍关节**：
   - `racket_grip`：标注在运动员持拍手与球拍的接触点
   - `racket_head`：标注在球拍拍面的几何中心
   - 如果球拍不可见，跳过这两个点
5. **精度要求**：关节点偏差不超过关节本身大小的 50%

```
head_top → head_center → chin → neck

```

## 3. 动作分类标注规范

### 3.1 动作类型（18 类）

| 动作代码 | 中文名 | 描述 |
|---------|--------|------|

### 3.2 动作阶段（6 阶段）

| 阶段代码 | 中文名 | 描述 |
|---------|--------|------|
| preparation | 准备阶段 | 判断来球、准备移动 |
| backswing | 引拍阶段 | 球拍向后引拍蓄力 |
| forward_swing | 挥拍阶段 | 球拍向前挥出 |
| contact | 击球瞬间 | 球拍与球接触的瞬间 |
| follow_through | 随挥阶段 | 击球后的惯性挥拍 |

### 3.3 动作质量评级

| 评级代码 | 中文名 | 描述 |
|---------|--------|------|
| standard | 标准动作 | 符合教科书标准的规范动作 |
| acceptable | 可接受 | 有小瑕疵但整体可接受 |
| needs_correction | 需纠正 | 存在明显技术错误 |

### 3.4 分类标注原则

1. **动作类型**：以当前帧最主要的动作为准，每帧只选一个类型
2. **动作阶段**：根据运动员的肢体姿态判断所处阶段
3. **质量评级**：
   - 标准动作：重心稳定、动作流畅、发力协调
   - 可接受：有轻微问题但不影响击球效果
   - 需纠正：明显的技术问题（如握拍不正确、重心不稳等）
4. **不确定时**：优先选择最接近的类别，并在备注中说明

## 4. 标注工作流程

### 4.1 流程概述

```
任务分配 → 标注员标注 → 自核 → 组长核对 → 专家终审 → 锁定
```

### 4.2 各环节职责

| 环节 | 负责角色 | 工作内容 |
|------|---------|---------|
| 标注 | 学生标注员 | 在 Label Studio 中完成骨架标注和动作分类 |
| 自核 | 学生标注员 | 自己检查标注结果，确认无误后提交 |
| 组长核对 | 标注组长 | 抽检标注质量，通过或打回 |
| 专家终审 | 标注专家 | 最终审核，确保标注符合规范 |

### 4.3 打回处理

- 打回时需注明具体问题
- 标注员根据反馈修正后重新提交
- 常见打回原因：关节点位置偏差、动作类型分类错误、遗漏关节点

## 5. 质量控制

### 5.1 一致性检查

- 连续帧之间关节点位置应连续变化，不应出现跳跃
- 同一动作序列内，动作阶段应符合时间顺序

### 5.2 常见错误

1. 左右关节点标反
2. 关节点标注在衣服边缘而非关节中心
3. 球拍点标注在拍柄而非握柄接触处
4. 将准备姿势误标为击球瞬间
5. 忽略反手动作的识别

### 5.3 标注质量指标

- 关节点位置精度：< 5 像素偏差
- 动作分类准确率：> 95%
- 标注完成率：按时完成分配任务的 100%
